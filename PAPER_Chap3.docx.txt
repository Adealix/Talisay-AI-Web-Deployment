CHAPTER 3
Research Methodology
RESEARCH DESIGN
        The study utilized developmental and experimental research method through the Agile software development model that was used to design, develop, and test a Machine Learning-Based Prediction of Seed-to-Oil Conversion Ratios in Talisay (Terminalia catappa) Fruits with Morphological Feature Analysis. Agile model was selected because of its innovative and short lived approach, in which it can be planned, developed, tested, and improved as the system progresses through its lifetime. The first step involved literature research and analysis of the problem and subsequently, the development of the system requirements was done in short functions known as sprints. Specific sprints were conducted in relation to dataset preparation, image processing, training of the Machine Learning model, morphological feature extraction and user interface refinement. At each iteration, to recognize accuracy and reliability constant testing and evaluation were made by using typical performance measures of the Machine Learning like the Mean Absolute Error (MAE) and Root Mean Square Error (RMSE). The feedback obtained after each cycle was used to improve the functionality of the system and the model performance, whereby the resulting system will eventually be an efficient instrument in data-driven predictions of seed-to-oil conversion ratio of Talisay fruits without any tests being destructive.


RESEARCH INSTRUMENT
        The primary research instrument used in this study is the Machine Learning–Based Prediction of Seed-to-Oil Conversion Ratios in Talisay (Terminalia catappa) Fruits using Morphological Feature Analysis developed by the researchers. This system functions as a tool for collecting, processing, and analyzing Talisay fruit image data. It incorporates image processing techniques to extract visual features and a trained machine learning model to identify its maturity and measure oil yield. The system was also used to generate details results,  such as accuracy and prediction reliability, which served as the basis for evaluating the effectiveness of the proposed solution. Supporting instruments include a labeled Talisay fruit image dataset obtained from reliable sources, used to train and evaluate the performance of the Machine Learning models


RESEARCH RESPONDENTS
        Farmers, harvesters, and Talisay tree growers will also form a part of the sample since they are the most targeted users of the proposed system. These experts have been chosen as they are the ones dealing with cultivation, collection, and process of Talisay fruits, and are thus the ones who will gain most in the management of an efficient and non-damaging reason to identify high-yield seeds, which produces oil. In order to obtain valid results of evaluation, the respondents must have basic mobile or computer literacy to be able to navigate through the system and evaluate its functionality as a decision-support tool. The effectiveness of the system in predicting the ratio of seed to oil conversion will be measured using their involvement, and it will provide the justification to apply the system practically in enhancing efficiency and minimizing waste in Talisay seed evaluation and agricultural resource management.

            * LEVEL 2:
  

Figure 7: Data Flow Diagram - Level 2: Machine Learning–Based Prediction of Seed-to-Oil Conversion Ratios in Talisay Fruits using Morphological Feature Analysis
        When the morphological features are passed on from the Scan and Image Processing module, the features are passed on to the Feature Normalization sub-process, which normalizes the features by scaling the numerical data (length, width, and mass). Once the normalization of the features has taken place, the features are passed on to the Model Inference Engine sub-process, which uses the machine learning algorithms (Random Forest and Gradient Boosting) to predict the oil yield by applying the algorithms to the features and arriving at a predicted value. Once the predicted value has been arrived at, the predicted value is passed on to the Post-Processing and Validation sub-process, which uses the algorithms to validate the predicted value within the range of 45% - 65% and to arrive at other information like the confidence intervals of the predicted value, based on the performance of the model. Once the predicted value and the confidence intervals have been arrived at, the predicted value and the confidence intervals are passed on to the user interface and are stored in the ML Analysis Results data store.


















            * ENTITY RELATIONSHIP DIAGRAM


  































Figure 8: Entity Relationship Diagram: Machine Learning–Based Prediction of Seed-to-Oil Conversion Ratios in Talisay Fruits using Morphological Feature Analysis
        The system uses a document-based NoSQL database structure (MongoDB) to design the database of the system, which procures the capability of effectively storing and managing all information of Talisay fruit scanning and oil yield prediction. The database is organized into five primary collections, each serving a particular purpose in the overall data architecture.
The central collection for user management is the “User” collection. It stores authentication details such as email (unique), passwordHash for secure login, role to differentiate between User and Admin, isVerified status, and profile information including firstName, lastName, phone, address, and avatar. Each user document also contains pushTokens for notification delivery and notificationSettings for preference management. This consolidated design eliminates the need for separate account, session, and profile tables by embedding related data within a single document.
“Prediction” collection stores the results of individual Machine Learning analyses. Each prediction document references a userId and contains the predicted oil yield ratio, confidence scores, color classification probabilities, dimensional measurements (length, width, kernel mass, whole fruit weight), and the model version used for inference.
“History” collection serves as the core transactional store for scanning activities. Each document records the userId, analysisType (single, comparison, or multi_fruit), imageUri pointing to cloud-stored images via Cloudinary, category classification, and for multi-fruit scans, the fruitCount, colorDistribution, and averageOilYield across all detected fruits.
“ForumPost” collection enables community interaction within the platform. Each document contains author reference, title, content, optional images array, a likes array tracking user engagements, and an embedded comments subdocument array where each comment contains its own author reference and message text. This embedded subdocument pattern avoids the need for a separate comments collection.
“Notification” collection manages system notifications, storing recipient references, notification type, message content, read status, and timestamps.
This document-based NoSQL design ensures flexibility and scalability. By embedding related data within documents rather than spreading it across multiple relational tables, the system achieves efficient read operations and a natural data model that aligns with how the application accesses information. The schema-flexible nature of MongoDB also allows the system to evolve its data structures as new features are developed for the Talisay seed oil prediction platform.
